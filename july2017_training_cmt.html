What do we use to determine who needs to be notified?
An event can be raised automatically by a trigger or manually by an engineer.
There are several event types.
## Workflow event Raised by Workflow Manager during a failed execution of an activity
## Quetex event Raised by stalled quetexes waiting for a resource to finish
## DBL event Raised by DBL keys signifying delayed updates
## JAMS event Raised by failed JAMS jobs
## Generic event Raised by Disk Alert, Generic Tool Failure, etc
## Manual event Raised manually by a stakeholder
Let's take a [look](http://cmt.prod.factset.com/website/#/events)!
## Event Actions ACTION | INTENT --- | --- `Page` | Notify a product stakeholder `Escalate` | Notify next level in the alert model `Raise` | Fire an event `Ack` | Act of investigating an event `Resolve` | Successful completion of an event
